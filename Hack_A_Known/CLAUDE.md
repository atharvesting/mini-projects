# CLAUDE.md — AI Assistant Instructions for Hack_A_Known

## Project Summary

A static, read-only Next.js web app that displays hackathons fetched from a Notion database. Users can toggle between a table view and a Gantt (timeline) view. Clicking a row opens a slide-over detail drawer.

---

## Hard Rules

- **JavaScript only. No TypeScript.** No `.ts` or `.tsx` files anywhere. Use `.js` and `.jsx` exclusively.
- **No `gantt-task-react`.** It is abandoned (last published 2021) and React 19 incompatible. The Gantt view is built as a custom CSS timeline component using Tailwind and `date-fns`.
- **Never prefix `NOTION_TOKEN` with `NEXT_PUBLIC_`.** This would expose the secret to the browser bundle.
- **Do not use `output: 'export'` in `next.config.js`.** It disables ISR entirely.
- **Do not use `export const runtime = 'edge'` on the main page route.** It breaks ISR.
- Prefer Server Components for data display. Use `"use client"` only where interactivity is required (view switcher, drawer, sort state).

## Key Patterns

### Fetching Notion Data (Server-side only — `lib/notion.js`)

Use `unstable_cache` to wrap the Notion SDK call. The SDK does not use native `fetch`, so Next.js cannot intercept and cache it automatically.

```js
import { unstable_cache } from 'next/cache'
import { Client } from '@notionhq/client'

const notion = new Client({
  auth: process.env.NOTION_TOKEN,
  timeoutMs: 10000, // Prevent build hangs if Notion API is slow
})

export const getHackathons = unstable_cache(
  async () => {
    const response = await notion.dataSources.query({
      data_source_id: process.env.NOTION_DATABASE_ID,
    })
    return response.results.map(normalizeHackathon)
  },
  ['hackathons'],
  { revalidate: 3600, tags: ['hackathons'] }
)
```

### Normalizing a Notion Page to a Hackathon Object

```js
function normalizeHackathon(page) {
  return {
    id: page.id,
    name: page.properties.Name.title[0]?.plain_text ?? 'Untitled',
    startDate: page.properties.Duration.date?.start ?? null,
    endDate: page.properties.Duration.date?.end ?? null,
    tags: page.properties.Tags.multi_select.map((t) => t.name),
    status: page.properties.Status.select?.name ?? null,
    url: page.properties.URL.url ?? null,
    description: page.properties.Description.rich_text[0]?.plain_text ?? '',
  }
}
```

### Hackathon Status Logic (`lib/dateUtils.js`)

```js
import { isAfter, isBefore, isToday, parseISO } from 'date-fns'

export function getStatus(startDate, endDate) {
  const today = new Date()
  const start = parseISO(startDate)
  const end = parseISO(endDate)
  if (isAfter(start, today)) return 'future'
  if (isBefore(end, today) && !isToday(end)) return 'past'
  return 'ongoing'
}
```

### Tailwind CSS v4 Setup — Key Differences vs v3

- `postcss.config.mjs` must use `@tailwindcss/postcss` (not `tailwindcss` directly).
- `globals.css` uses `@import "tailwindcss"` instead of `@tailwind base/components/utilities`.
- Font variables from `next/font` are injected in the `@theme {}` block in `globals.css`, not in `tailwind.config.js`.
- `shadow-sm` → `shadow-xs` | `rounded-sm` → `rounded-xs` | `focus:outline-none` → `focus:outline-hidden`

### shadcn/ui JS Mode

`components.json` must have `"tsx": false`. The CLI generates `.jsx` files. Run:

```bash
npx shadcn@latest add table sheet badge tabs button
```

### Client Components for Interactivity

Any component using `useState`, `useEffect`, event handlers, or shadcn interactive primitives needs:

```js
'use client'
```

at the very top of the file. The main `page.js` is a Server Component that passes fetched data as props down to client components.
